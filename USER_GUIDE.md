## הקדמה

MTK Firmware Editor Pro היא תוכנה מקצועית לעריכת קושחות Android עבור מכשירים מבוססי MediaTek chipsets. התוכנה מאפשרת חילוץ, עריכה, ויצירת חבילות קושחה מוכנות לצריבה באמצעות SP Flash Tool.

### למי מיועדת התוכנה?
- מפתחי ROM מותאמים אישית
- טכנאי תיקון מכשירים
- חובבי התאמה אישית של Android
- חוקרים ופיתוח

---

## דרישות מקדימות

### תוכנה נדרשת

#### 1. Python 3.8+
```
הורדה: https://www.python.org/downloads/
התקנה: סמן "Add Python to PATH" בזמן ההתקנה
```

#### 2. SP Flash Tool
```
הורדה: חפש "SP Flash Tool download" או מאתר MediaTek הרשמי
גרסה מומלצת: v5.2032 ומעלה
```

#### 3. MTK USB Drivers
```
הורדה: "MediaTek PreLoader USB VCOM Drivers"
הערה: נדרש כדי שהמחשב יזהה את מכשיר ה-MTK
```

### חומרה נדרשת
- מחשב Windows 10/11
- יציאת USB פעילה
- כבל USB איכותי (מומלץ כבל מקורי)
- שטח דיסק: 10GB לפחות (עבור עבודה עם קושחות)

### מכשיר תואם
- מכשיר Android מבוסס MTK chipset
- סוללה טעונה לפחות 50%
- USB Debugging מופעל (אם אפשרי)

## תהליכי עבודה

### תהליך 1: חילוץ קושחה מהמכשיר

#### שלב 1: הכנת המכשיר
```
אופציה א - מכשיר פועל:
1. הפעל USB Debugging
2. חבר למחשב
3. אשר debugging prompts

אופציה ב - מכשיר לא נדלק:
1. כבה את המכשיר לחלוטין
2. הסר סוללה (אם ניתן) והחזר
3. המכשיר יזוהה אוטומטית בחיבור
```

#### שלב 2: זיהוי מכשיר
```
1. הפעל את התוכנה (run.bat או python main.py)
2. לחץ "Detect Device"
3. המתן לזיהוי
4. בדוק ש-Device Panel מציג "Connected"
```

#### שלב 3: חילוץ מחיצות
```
1. לחץ File → Open Workspace
2. בחר תיקייה ריקה לעבודה
3. לחץ "Extract Partitions"
4. המתן לסיום (עשוי לקחת 10-30 דקות)
5. בדוק שכל המחיצות חולצו בהצלחה
```

---

### תהליך 2: עריכת קושחה

#### A. שינוי Boot Logo

##### שלב 1: הכן תמונה
```
דרישות תמונה:
- פורמט: PNG, JPG, או BMP
- רזולוציה: התאם לרזולוציית המכשיר
  (לדוגמה: 1080x1920 עבור FHD)
- גודל: לא יותר מ-5MB
```

##### שלב 2: החלת לוגו
```
1. עבור ללשונית "Editor"
2. תחת "Boot Logo Editor":
   - לחץ "Browse"
   - בחר את התמונה
   - לחץ "Apply Logo"
3. בדוק הודעת הצלחה
4. צפה בתוצאה ב-"Preview" tab
```

#### B. עריכת Build Properties

##### שינויים נפוצים:
```properties
# שינוי שם מכשיר
ro.product.model=My Custom Phone
ro.product.brand=CustomBrand

# שינוי מידע גרסה
ro.build.display.id=CustomROM v1.0
ro.build.description=Custom Build

# תכונות נוספות
ro.debuggable=1
persist.sys.usb.config=mtp,adb
```

##### שלב ביישום:
```
1. בלשונית "Editor"
2. תחת "Build Properties Editor"
3. הכנס properties בפורמט key=value
4. לחץ "Apply Properties"
5. בדוק ביומן השינויים
```

### תהליך 4: יצירת חבילה לצריבה

#### שלב 1: יצירת Scatter File
```
1. לחץ Tools → Generate Scatter File
2. בחר שם ומיקום לקובץ
3. שמור (שם מומלץ: MT{CHIP}_Android_scatter.txt)
```

#### שלב 2: ייצוא חבילה
```
1. לחץ Tools → Export Modified Firmware
2. בחר תיקיית יעד
3. המתן ליצירת החבילה
4. התוכנה תיצור:
   - כל קבצי ה-partitions
   - קובץ scatter
   - README.txt
   - FLASH_INSTRUCTIONS.txt
   - checksums.md5
```

### תהליך 5: צריבה למכשיר

⚠️ **אזהרה**: תהליך זה עלול להזיק למכשיר אם לא מבוצע נכון!

#### הכנה לצריבה
```
1. טען סוללה ל-50% לפחות
2. גבה את כל המידע החשוב
3. וודא שיש לך גיבוי של הקושחה המקורית
4. הכן כבל USB איכותי
```

#### שימוש ב-SP Flash Tool

##### שלב 1: טעינת Scatter
```
1. פתח SP Flash Tool
2. לחץ "Scatter-loading"
3. בחר את קובץ ה-scatter שנוצר
4. בדוק שכל הקבצים נמצאו (סימונים ירוקים)
```

##### שלב 2: בחירת מצב הורדה
```
מצבים זמינים:
- Download Only: צריבה מלאה (מומלץ)
- Firmware Upgrade: עדכון חלקי
- Format + Download: מחיקה וצריבה
```

##### שלב 3: צריבה
```
1. לחץ "Download"
2. כבה את המכשיר לחלוטין
3. חבר למחשב דרך USB
4. המכשיר יזוהה ויתחיל תהליך הצריבה
5. אל תנתק! המתן למעגל ירוק
6. נתק ואתחל את המכשיר
```

##### זמני אתחול צפויים:
```
- אתחול ראשון: 5-15 דקות
- אתחולים הבאים: רגיל (1-2 דקות)
```

## פתרון בעיות

### בעיה: מכשיר לא מזוהה

#### פתרון 1: בדיקת Drivers
```powershell
# בדוק התקנת drivers
1. פתח Device Manager
2. חפש "Ports (COM & LPT)"
3. חפש "MediaTek PreLoader USB VCOM"
4. אם יש סימן קריאה צהוב - הדרייברים לא תקינים
```

#### פתרון 2: יציאת USB אחרת
```
- נסה יציאת USB אחורית (במחשב שולחני)
- נסה USB 2.0 במקום 3.0
- השתמש בכבל מקורי
```

#### פתרון 3: הכנס למצב Download ידנית
```
MTK Download Mode Combinations (תלוי במכשיר):
- Volume Down + Power + חבר USB
- Volume Up + Volume Down + חבר USB
- רק חיבור USB כשמכשיר כבוי
```

### בעיה: SP Flash Tool - שגיאה אדומה

#### שגיאות נפוצות:

##### BROM ERROR: S_FT_ENABLE_DRAM_FAIL (4032)
```
פתרון:
- קובץ preloader לא תואם
- השתמש ב-preloader מקורי של המכשיר
```

##### S_DA_INT_RAM_ERROR (4050)
```
פתרון:
- בעיה בקובץ DA
- הורד גרסה אחרת של SP Flash Tool
```

##### S_BROM_CMD_STARTCMD_FAIL (5054)
```
פתרון:
- כבל USB לא תקין
- החלף כבל
- יציאת USB אחרת
```

##### PMT Changed (6010)
```
פתרון:
- מבנה מחיצות השתנה
- בחר "YES" אם אתה בטוח
- או השתמש בפריסה מקורית
```

---

### בעיה: מכשיר לא נדלק אחרי צריבה

#### שלב 1: לולאת אתחול (Bootloop)
```
פתרון:
1. כנס ל-Recovery Mode:
   Volume Up + Power
2. בצע Factory Reset / Wipe Data
3. אתחל מחדש
```

#### שלב 2: עדיין לא נדלק
```
פתרון:
1. חזור לSP Flash Tool
2. צרוב שוב את הקושחה המקורית (גיבוי)
3. או צרוב Stock ROM של המכשיר
```

#### שלב 3: Hardbrick
```
אם שום דבר לא עובד:
- המכשיר ב-hardbrick
- דרוש תיקון מקצועי או JTAG
```

## שאלות נפוצות

### ש: האם התוכנה תומכת במכשיר שלי?
**ת**: התוכנה תומכת במכשירי MTK. בדוק את ה-chipset של המכשיר שלך:
```
Settings → About Phone → Processor
או
השתמש באפליקציה כמו CPU-Z
```

### ש: האם אני צריך root?
**ת**: לא, אך USB Debugging עוזר לזיהוי טוב יותר.

### ש: כמה זמן לוקח חילוץ קושחה?
**ת**: תלוי בגודל המחיצות ובמהירות USB. בדרך כלל 10-30 דקות.

### ש: האם אפשר לבטל שינויים?
**ת**: כן! לכן חשוב ליצור גיבוי לפני עריכה. פשוט צרוב את הגיבוי המקורי.

### ש: מה לעשות אם התוכנה קורסת?
**ת**: 
1. בדוק קובץ log ב-logs/
2. ודא שכל התלויות מותקנות
3. נסה להפעיל מחדש
4. הפעל כמנהל

### ש: האם זה בטוח?
**ת**: התוכנה בטוחה אם משתמשים בה נכון. הסיכונים הם:
- צריבת קושחה לא תואמת
- ניתוק בזמן צריבה
- מחיקת מחיצות קריטיות

### ש: איך יוצרים גיבוי?
**ת**: בשלב חילוץ המחיצות, שמור את כל הקבצים. זה הגיבוי שלך!
